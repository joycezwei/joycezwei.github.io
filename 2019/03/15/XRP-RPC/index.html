简介

1.JSON-RPC

您可以使用任何HTTP客户端（例如Poster for Firefox或Postman for Chrome）将JSON-RPC调用为rippled服务器。大多数编程语言都有一个用于创建HTTP请求的库。

请求格式
要发出JSON-RPC请求，请将HTTP POST请求发送到服务器正在侦听JSON-RPC连接/的端口和IP上的根路径（）rippled。您可以使用HTTP / 1.0或HTTP / 1.1。如果你使用HTTPS，你应该使用TLS v1.2。出于安全原因，rippled 不支持 SSL v3或更早版本。
始终包含一个Content-Type包含值的标题application/json。
如果您打算提出多个请求，请使用Keep-Alives，以便您不必关闭并重新打开请求之间的连接。

将请求主体作为具有以下属性的JSON对象发送：
将该命令放在顶层”method”字段中
包括顶级”params”字段。这个字段的内容应该是一个只包含一个嵌套JSON对象并包含该命令所有参数的单项数组。
响应也是一个JSON对象。

公共服务器
目前，Ripple（公司）在以下地区维护一套公共JSON-RPC服务器：

Domain Port Notes
s1.ripple.com 51234 General purpose server
s2.ripple.com 51234 Full-history server

这些公共服务器不用于持续或商业用途，并且可能随时变得不可用。对于常规使用，您应该运行自己的波纹服务器或与您信任的人签订合同。

2.API方法

Websocket和JSON-RPC API的API方法由命令名称定义，并分为公用命令和管理命令。
https://developers.ripple.com/public-rippled-methods.html
公共命令不一定适用于普通公众，但是它们被连接到服务器的任何客户端使用。（将公共命令看作是运行服务器的组织的成员或客户，而管理命令用于负责保持服务器运行的人员。）公共命令包括诸如检查分类账状态，查找连接用户的路径以及提交交易等等。另一方面，管理命令仅用于受信任的服务器操作员，并包含用于管理，监视和调试服务器的命令。
https://developers.ripple.com/admin-rippled-methods.html
使用这些管理API方法直接与ripple服务器通信。管理方法仅适用于负责保持服务器运行的可信人员。管理方法包括用于管理，监视和调试服务器的命令。
仅当您连接到rippled.cfg文件标识为admin的主机和端口上的rippled时，Admin命令才可用。默认情况下，命令行客户端使用管理连接。有关连接到波纹的更多信息，请参阅波纹API入门。

1.data api v2

https://developers.ripple.com/data-api.html
Ripple Data API v2提供对XRP Ledger中更改信息的访问，包括事务历史记录和已处理的分析数据。 此信息存储在专用数据库中，该数据库可释放rippled 服务器以保留较少的历史分类帐版本。 Data API v2还充当XRP图表和ripple.com等应用程序的数据源。
Ripple在以下地址提供尽可能完整的事务记录的Data API实时实例：
https://data.ripple.com
