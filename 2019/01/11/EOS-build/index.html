在ubuntu搭建EOS环境

虚拟机 ubuntu16.04 内存放到8g
项目官网https://github.com/EOSIO/eos，关于如何构建官方网站https://developers.eos.io/eosio-nodeos/docs

deb安装

先下载

下载报错404，直接在官网下载

没有运行nodeos

autobuild script

换一种方式装，在官网有三种方式：Autobuild Script，Docker-compose，Manually Build，这里使用第一种方式
—recursive表示下载其子模块

如果之前没有用—recursive，可以之后用下面指令下载

全平台系统要求 7GB RAW和20 GB disk，自动脚本构建，开发者建议从源码开始构建
踩坑可以参考https://www.jianshu.com/p/13c7bddacfd9
cd eos
第一次build报错，什么unable to retrieve …,然后我再试了一遍没有这个报错，
./eosio_build.sh //1,会下载一些安装包
报错二：

没搞定这个问题，重装了虚拟机重新装,重装后更新所有软件又遇到了循环登陆的问题，重新装了显卡还是有问题,干脆升级到ubuntu18,但是升级要好久，所以重新装一个虚拟机不升级然后装
做了一个实验，升级到18.04后没法安装parallel tool，应该是我的parallel版本低的原因，我这个版本支持的是Ubuntu16版本
sudo ./eosio_install.sh
在build后

真是可喜可贺，然后install也很快

keosd、nodeos、cleos在/usr/local/eosio/bin目录下

docker

或者选择docker安装，先安装docker https://docs.docker.com/install/linux/docker-ce/ubuntu/#set-up-the-repository
然后按照官网步骤来

运行本地EOS节点

这是EOS官网的组件架构关系图，描述了不同组件之间的关系：

  • nodeos - 服务器端的区块链节点组件
  • keosd - 钱包管理组件
  • cleos - 操作 nodeos 和 keosd 的命令行工具
    账户、钱包、权限，这部分的内容参考自https://www.jianshu.com/p/8aa9db8b4f7a

账户里有：公钥，合约，eos的账户名可以自定义
钱包里有：私钥，每个钱包保存一个私钥和打开钱包的密码
权限：每个账户有3种权限，owner最高权限，可以修改其他权限、active可以进行转账等操作，但是不能修改权限、recovery可以用于恢复账户使用权
在脚本构建后

接下来创建账号的话需要开启nodeos

会不断产出区块，因为是本地节点运行，所以从1开始，作为单独的区块生产者，数据在～/.local/share/eosio/nodeos/data
